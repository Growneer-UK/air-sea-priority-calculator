# 新品空运优先级计算器

用于首批测款时，按“每 1 元空运新增成本能够提前释放多少毛利潜力”排列 SKU 的空运优先级，并计算每个产品的空运/海运边界计费重。

## 海运成本口径

公司原本就会持续发整柜，不代表测款 SKU 的海运成本为零。本工具采用完整成本口径：每个 SKU 的海运成本，按其占用 40HQ 体积或载重的比例，从整柜约 US$8,000 的整体总费用中计算。

默认取体积占比与重量占比中较高者，避免低估轻抛货或重货对柜内容量的占用：

```text
整柜人民币成本 = 40HQ整体总费用（美元）× 汇率
体积占比 = SKU整批海运体积 ÷ 40HQ有效装载体积
重量占比 = SKU整批海运毛重 ÷ 40HQ有效装载重量
SKU海运成本 = 整柜人民币成本 × MAX（体积占比，重量占比）
              + 单SKU海运固定费用
```

US$8,000 是默认值，可按公司的实际整柜总成本调整。默认汇率为 6.8，因此默认整柜人民币成本为 ¥54,400。有效装载体积、有效装载重量、汇率和单 SKU 海运固定费用也都可以在页面中修改。页面统一把公式算出的金额称为“海运成本”；“体积占比”或“重量占比”只说明该成本的计算依据。

逐 SKU 取体积占比和重量占比中的较高值，是偏保守的单 SKU 管理决策口径。它能避免低估某个轻抛货或重货的柜内容量占用，但所有 SKU 的计算结果相加可能高于整柜实际费用，因此不能直接用于整柜财务对账。需要严格对账时，应由财务统一一种计费基准，或对全部 SKU 的权重做归一化。

## 产品点口径

- 1 产品点 = 产品成功后年毛利潜力 ¥120,000
- 新增产品默认 0.1 点，即年毛利潜力 ¥12,000，可逐个修改
- 允许填写小数，例如 0.5 点代表年毛利潜力 ¥60,000
- 所有产品应使用公司统一的毛利定义

产品点表达的是产品测试成功、做成正常大货后的年毛利潜力，不是首批测款利润，也没有乘入成功率。首次测款采购价可能与正常大货采购价差异很大，因此首次采购成本不直接进入产品点，也不直接决定 AAI 排名；它仍可作为实际现金预算或人工决策的补充条件。

## 核心公式

```text
年毛利潜力 = 产品点 × ¥120,000
提前毛利潜力 = 年毛利潜力 × 有效提前天数 ÷ 365

空运新增成本 = 空运成本 − 海运成本
AAI = 提前毛利潜力 ÷ 空运新增成本
```

AAI 越高，说明同样多花 1 元选择空运，能够提前释放的毛利潜力越大，因此越应该排在空运队列前面。

默认优先级分档为：

- AAI ≥ 3：高优先
- 2 ≤ AAI < 3：中优先
- 1.5 ≤ AAI < 2：达到最低入选线，可纳入空运方案
- AAI < 1.5：未达到最低入选线，建议海运

高优先内部线与中优先内部线只负责给已经计算出的 AAI 分档，方便排序和阅读；它们不决定空海运边界。空海运边界只由“当前最低入选线”决定，默认使用 AAI=1.5。三条线都可以回调，修改最低入选线后，页面中的边界总成本、边界计费重和运输建议必须同步更新。

AAI 是“产品测试成功”情景下的优先级指标，不是经过成功率加权的财务收益率。空运与海运不会改变同一产品本身的成功率，因此比较该产品的两种运输方式时，不需要因空运成本更高而再给空运单独乘一个成功率。当前模型也不比较不同产品之间的成功率差异。

如果公司以后统一加入成功率 `p`，则所有产品的提前毛利潜力和 AAI 都会乘以 `p`，排名不变；为了保持当前入选决策不变，最低入选线也必须同时乘以 `p`。例如统一成功率为 25% 时，成功情景最低线 1.5 对应的概率加权最低线是 `1.5 × 25% = 0.375`。只给 AAI 乘成功率、却仍拿它与原来的 1.5 比较，会人为减少空运入选数量。若要使用每个产品不同的成功率，则属于另一套“概率加权排名”，需要为每个 SKU 单独提供可信的成功率。

## 重量、体积与空运成本

重量和体积会同时影响空运成本与海运成本，因此都与最终结论有关。

```text
空运体积重 = 外箱长 × 宽 × 高 ÷ 抛重除数
空运计费重 = MAX（整批实重，整批体积重），再按货代规则进位
空运成本 = 空运计费重 × 空运单价 + 空运固定费用
```

轻抛货通常会因体积重提高空运计费重；重货则可能因重量占比较高而增加海运成本。页面会自动比较两种运输方式，不应只用一个与体积、重量无关的固定公斤数作为所有产品的边界。

## 空海运边界计费重

边界表示“AAI 刚好达到当前最低入选线”的位置。设最低入选线为 `T`，则：

```text
边界允许的空运新增成本 = 提前毛利潜力 ÷ T
边界空运总成本 = 海运成本 + 提前毛利潜力 ÷ T
理论空运边界计费重
=（边界空运总成本 − 空运固定费用）÷ 空运单价
```

- 实际空运总成本 ≤ 边界空运总成本，同时实际计费重 ≤ 边界计费重：达到最低入选线，优先考虑空运
- 实际空运总成本 > 边界空运总成本，同时实际计费重 > 边界计费重：未达到最低入选线，优先考虑海运

其中：

```text
实际空运总成本 = 实际空运计费重 × 空运单价 + 空运固定费用
```

成本比较与公斤数比较是同一判断的两种表达，不能只显示公斤数。结果区应明确展示实际空运总成本，并显示“实际空运总成本 ≤ / > 边界空运总成本”；公斤数区域同时显示“实际计费重 ≤ / > 边界计费重”。

公式算出的公斤数是连续理论边界。若货代按 0.5kg 或 1kg 进位，最后一个真正可执行的计费重可能略低于理论边界，因此最终结论以实际空运总成本和 AAI 比较为准。

以默认 0.1 产品点、提前 50 天、空运费 ¥40/kg 为例，提前毛利潜力为：

```text
¥12,000 × 50 ÷ 365 = ¥1,643.84
```

“基础边界（暂按 SKU 海运成本 ¥0）”就是暂时不代入具体 SKU 的体积和毛重，只用于给出一个通用参考值；它不代表公司实际海运费为零，也不能直接作为某个 SKU 的最终边界。基础边界使用最低入选线，而不是固定使用 AAI=1。

按默认最低入选线 AAI=1.5、空运费 ¥40/kg、空运固定费用 ¥0 计算：

```text
基础边界允许的空运新增成本 = ¥1,643.84 ÷ 1.5 = ¥1,095.89
基础边界空运总成本 = ¥0 + ¥1,095.89 = ¥1,095.89
基础边界计费重 =（¥1,095.89 − ¥0）÷ ¥40/kg = 27.40kg
```

高优先线 3 和中优先线 2 与这个边界没有直接关系；只有最低入选线 1.5 进入上述公式。具体 SKU 的准确边界还要加上其自动计算的海运成本，所以会随整批体积和整批毛重变化，并不都是 27.40kg。

使用边界模块的默认整批体积 0.1m³、整批毛重 20kg、40HQ 有效体积 68m³、有效载重 26,000kg 计算时，体积占比高于重量占比：

```text
整柜人民币成本 = US$8,000 × 6.8 = ¥54,400
SKU海运成本 = ¥54,400 × MAX（0.1 ÷ 68，20 ÷ 26,000）= ¥80.00
边界允许的空运新增成本 = ¥1,643.84 ÷ 1.5 = ¥1,095.89
边界空运总成本 = ¥1,095.89 + ¥80.00 = ¥1,175.89
边界计费重 = ¥1,175.89 ÷ ¥40/kg = 29.40kg
```

在默认空运抛重除数 6,000、按 0.5kg 向上进位时，同一批货的实际空运计费重也会由页面自动填写：

```text
体积重 = 0.1m³ × 1,000,000 ÷ 6,000 = 16.67kg
实重与体积重取较高者，再按 0.5kg 向上进位 = 20.00kg
```

因此默认边界模块中，实际空运总成本为 `20.00kg × ¥40/kg + ¥0 = ¥800.00`，低于边界空运总成本 ¥1,175.89；实际计费重 20.00kg 也低于理论边界 29.40kg。当前 AAI 为 `¥1,643.84 ÷（¥800 − ¥80）= 2.28`，达到默认最低入选线且属于中优先，建议空运。按0.5kg进位时，29.40kg理论边界对应的可执行上限为29.00kg。用户仍可直接覆盖自动填写的实际计费重，以货代给出的最终计费重为准；修改体积、毛重、抛重除数或进位单位后，自动值会随之更新。

若使用先前截图中的空运单价 ¥45/kg、固定费用 ¥10，并手工把实际计费重改为 17.00kg，则页面应展示：

```text
实际空运总成本 = 17.00kg × ¥45/kg + ¥10 = ¥775.00
边界空运总成本 = ¥1,095.89 + ¥80.00 = ¥1,175.89
边界计费重 =（¥1,175.89 − ¥10）÷ ¥45/kg = 25.91kg
成本比较：¥775.00 < ¥1,175.89
公斤数比较：17.00kg < 25.91kg
结论：达到最低入选线 AAI=1.5，建议空运
```

## 默认三个示例

三个示例都使用 0.1 产品点和默认 50 天，测试采购价只用于预算。差异完全来自整批体积和实重：

| 示例 | 整批体积 | 实重 | 计费重 | 空运成本 | 海运成本 | AAI | 分档与方案 |
|---|---:|---:|---:|---:|---:|---:|---|
| 轻小件高优先 | 0.03m³ | 5kg | 5kg | ¥200.00 | ¥24.00 | 9.34 | 高优先，空运 |
| 中等重量中优先 | 0.12m³ | 20kg | 20kg | ¥800.00 | ¥96.00 | 2.33 | 中优先，空运 |
| 大体积抛重低优先 | 0.30m³ | 10kg | 50kg | ¥2,000.00 | ¥240.00 | 0.93 | 低优先，海运 |

三条阈值实际形成四个区间。这三个示例特意展示“高优先空运、中优先空运、低于入选线海运”三种主要情况；`1.5 ≤ AAI < 2` 的产品仍会显示“达到入选线”并可进入空运方案。

若沿用页面先前填写的空运单价 ¥45/kg、固定费用 ¥10，三个示例的 AAI 会分别变为 7.79、2.02、0.81，运输方案仍然是两个空运、一个海运。因此第二个示例在两套参数下都能稳定体现“中优先空运”。

页面提供独立的“边界计费重快速计算”模块。输入产品点、有效提前天数、整批海运体积和整批海运毛重后，模块会自动继承全局的整柜费用、汇率、有效装载容量、空运抛重规则与运价，并自动计算实际空运计费重。用户也可以手工覆盖该值。模块输出：

- SKU 海运成本及其计算依据（体积占比或重量占比）
- 对应当前最低入选线（默认 AAI=1.5）的空运总成本与边界计费重
- 当前实际空运总成本，以及它与边界空运总成本的大小比较
- 当前实际计费重对应的空运或海运建议

右侧结果区的每个指标都带有计算说明。电脑端将鼠标悬停在指标卡片上即可即时查看；键盘操作时将焦点移入卡片也会显示相同说明，便于核对取值来源和公式。

## 功能

- 输入每个 SKU 的产品点和有效提前天数（默认 50 天）
- 计算实重、体积重、计费重和抛重倍数
- 按体积占比或重量占比自动计算每个 SKU 的 40HQ 海运成本
- 输出 AAI 排名、AAI=1 盈亏平衡产品点和提前天数
- 为每个 SKU 计算独立的空运边界计费重
- 按空运新增预算或最多 SKU 数给出组合
- 支持 JSON 导入导出、CSV 结果导出、打印和本机自动保存
- “计算口径与填写说明”默认全部展开，便于共同核对
- 单文件 HTML，无后端，录入数据不会上传服务器
